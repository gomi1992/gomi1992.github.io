发表于 2019-10-18 更新于 2020-08-19 分类于 网站搭建 Disqus： Valine：

使用语雀搭建静态博客。

为什么

网上有 hexo 建 github 博客的教程，也有结合语雀和自动构建的教程。
但是在我实践一遍后发现了一些问题，这就是我写这篇文章的原因。
本文与其他教程不同之处：

  • 本文基于 hexo 官方教程，搭建环境更为简单
  • 解决部署到 github 主页（用户名.github.io）不成功的问题
  • 成功同步语雀私有文档到博客

建站

nodejs 的安装及 hexo 的安装，这里就不多做阐述，可以参考其他资料。
这里注意下，由于国内环境的原因，最好是使用国内的 npm 镜像源。

部署到 github pages

详细步骤参考https://hexo.io/zh-cn/docs/github-pages

坑 0–github pages

github pages 分为 user site 和 project site。
user site 是以 username.github.io 为项目名(username 为用户的用户名)。
project site 是项目站点，访问时会访问到 username.github.io/projectname。
我在使用 user site 建站时，发现我的站点无法访问，而使用 project site 建的就没有任何问题。

这是我 user site 的 github pages 设置界面。它很明确的说明了 user pages 会从 master 分支进行构建。
但它没有明确的说是谁来构建，这里 gh 用的是 jekyll 来构建相关网页。

这是我 project stie 的设置界面。project stie 是可以选择分支作为站点的。

那这个问题怎么解决呢？
在新建项目时网站源文件（即 md 文件及相关配置）存放在非 master 分支下，我使用 hexo 来存放我的网页源文件。
