JavaFX 手记 00，为什么我要用 JavaFX？！

起因

跟师兄一起干个项目，其中需要一个桌面客户端。
于是全职工具人的我就背锅了。

为什么选择 JavaFX 呢

  1. 对 java 比较熟
  2. 具有较好的移植性
  3. 在 windows 环境下不需要装 vs 就可以开发了
  4. 默认界面还看得过去
  5. 对多媒体支持比较好
  6. 有可视化的 UI 编辑工具
  7. 使用 XML 作为 UI 配置
  8. 快速上手
  9. 。。。

为了快速开发出产品原型，基于以上几点理由我选择了 JavaFX 作为开发平台。
但也因此遇到了很多坑。

JDK 1.8

为啥是 1.8？
这是我经理好多血泪的教训才下的决定。目前 JDK 有许多比 1.8 高的版本，而且 JDK 1.8 也基本结束了支持。
选择 JDK 1.8 主要有一下几点理由

  • 1.8 版本不要单独安装 JavaFX，1.8 以后的版本都要单独安装 JavaFX
  • 1.8 版本可以很轻松的导出成 jar 或者可执行文件

开发环境

  • JDK 1.8
  • IDEA，好用的 ide
  • SceneBuilder-8.5.0，配合 JDK 1.8 版本选择的

目的

本手记目的是记录项目开发过程中遇到的问题和解决方法。
